HTML, CSS y JavaScript

JavaScript es el lenguaje de programación que permite crear aplicaciones web y móviles a lo largo de múltiples tipos de dispositivos y navegadores web. Es un lenguaje que sigue la especificación ECMAscript y que es, junto a HTML y CSS, una de las tres tecnologías elementales en el mundo de la web. La mejor forma de entender JavaScript es viéndolo en acción. Se lo encuentra en cualquier web. Y, aunque el alcance de este lenguaje llega también a dispositivos móviles y otro tipo de "hardware", la web siempre es el punto clave de partida. Lo que ves en una página web es la combinación de este tridente de tecnologías: HTML, CSS y JavaScript, interpretados por el navegador web. La primera capa es HTML, la cual solo da estructura y significado al contenido que se muestre. Este, al ser un lenguaje estrictamente de marcado, carece de ciertas características de vista e interactividad. Para cuestiones de vista, existe la segunda capa en una página web, la cual es cubierta por CSS y, aunque también puede disponer de algunas opciones de interactividad, se enfoca principalmente en cómo se verá el contenido, es decir, en su presentación. Ya estructurado y bien presentado el contenido, entonces entra en acción la tercera capa, que es cubierta por JavaScript, un lenguaje interpretado que se ejecuta en el navegador e interactua perfectamente con las capas de HTML y CSS manipulando e interactuando con el contenido. Desde el más básico clic de un botón hasta una sofisticada animación o la invocación de datos y servicios remotos.

HTML determina la estructura, CSS la presentación y JavaScript el comportamiento, lo que hace la página.

JavaScript se ha vuelto la piedra angular en el desarrollo web contemporáneo. Hay dos razones principales por las que JavaScript se ha vuelto tan popular en el desarrollo web: su capacidad para renderizar contenido dinámico y su capacidad para realizar tareas asincrónicas.

Contenido web estático vs dinámico

Uno de los primeros beneficios de JavaScript fue su capacidad para manejar contenido dinámico. En los primeros días de la web, todas las páginas web eran estáticas, lo que significa que los datos mostrados en la página eran cargados por el navegador de antemano, y la página tenía que ser recargada para mostrar cualquier actualización. Los sitios web son dinámicos cuando son capaces de renderizar nuevos datos sin tener que recargar la página, y este es el tipo de comportamiento que JavaScript fue diseñado para hacer. Por ejemplo, gracias a JavaScript, podemos ver nuevos mensajes aparecer en un feed de redes sociales sin tener que actualizar la página.

Realización de tareas asincrónicas

Para que el usuario tenga una experiencia fluida al trabajar con un sitio web dinámico, éste debe ser capaz de realizar tareas de manera asincrónica. Muchos lenguajes de programación operan de manera sincrónica, lo que significa que una operación debe ser completada antes de que la siguiente operación comience.
