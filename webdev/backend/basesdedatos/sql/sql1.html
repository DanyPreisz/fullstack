Introducción a las bases de datos

Datos e Información

Los datos son hechos o valores sin procesar que no tienen un significado específico por sí mismos. Son elementos básicos y sin interpretación, que pueden ser números, palabras, mediciones, observaciones, descripciones, etc. Los datos pueden ser recolectados y almacenados, pero no proporcionan contexto o entendimiento.

Ejemplos de datos:

  • Números: 42, 7.89, 1001
  • Palabras: "rojo", "grande", "rápido"
  • Fechas: "2024-06-07"
  • Valores medidos: 25°C, 5m/s

La información es el resultado de procesar, organizar o estructurar datos de una manera que les confiere significado y contexto. La información es útil porque responde a preguntas y es comprensible, proporcionando conocimientos sobre un tema o fenómeno. Es el producto de analizar y contextualizar los datos.

Ejemplos de información:

  • "La temperatura actual es de 25°C" (contextualiza un dato de temperatura)
  • "El promedio de ventas mensuales en el último trimestre fue de 10,000 unidades" (analiza datos de ventas)
  • "El próximo examen es el 7 de junio de 2024" (proporciona contexto a una fecha)
  • "El color más popular entre los encuestados fue el rojo" (da significado a una colección de datos sobre preferencias de color)

La diferencia entre datos e información radica en el contexto y el procesamiento: Los datos son insumos crudos que necesitan ser procesados o analizados. la información es el resultado del procesamiento de los datos, lo que les otorga significado y utilidad para la toma de decisiones. Los datos se convierten en información cuando se procesan y organizan de una manera que es significativa y útil para quienes la reciben.

¿Qué es una base de datos?

Una base de datos es una colección de datos interrelacionados almacenados en conjunto, sin redundancias perjudiciales e innecesarias, con la finalidad de servir a una aplicación o más, de la mejor manera posible. Los datos se almacenan de forma independiente de los programas que los usan. Se emplean métodos bien determinados para incluir nuevos datos y para modificar o extraer los datos almacenados.

Es un conjunto de datos pertenecientes a un mismo contexto organizados para un propósito específico que. Una base de datos nos permite acceso, gestión y actualización de manera estructurada y eficiente.

Las bases de datos son sistemas esenciales que nos permiten guardar, organizar y recuperar información de manera eficiente. Son cruciales para operar y entender numerosas aplicaciones digitales en la vida cotidiana y de negocios. Aprender sobre ellas es fundamental porque facilitan la toma de decisiones basadas en datos y mejoran nuestra interacción con la tecnología.

Las bases de datos permiten:

  • Almacenar datos.
  • Acceder a los datos.
  • Agregar, modificar y eliminar datos.
  • Analizar datos.

Bases de datos o archivos planos

La simplicidad de tener un archivo de Excel, un archivo de texto txt o csv o de Google Sheets al alcance de la mano nos puede llevar a elegirlo para guardar los datos. Pero el uso de archivos tiene algunos inconvenientes.

Las bases de datos realizan chequeos de integridad, coherencia de los datos y redundancia que los archivos planos no realizan.

Sistemas de archivos planos o ficheros

  • Sist. orientados a los procesos
  • Ficheros específicos para aplicaciones
  • Redundancia de datos
  • Inconsistencia de datos
  • Pérdida de espacio físico
  • Dependencia física de datos
  • Dependencia lógica de datos

Sistemas de bases de datos

  • Sist. orientados a los datos (uso racional)
  • No son diseñados para una aplicación
  • Disminuir redundancia
  • Evitar inconsistencia
  • Mantener formato de datos
  • Independencia física
  • Independencia lógica
  • Compartir los datos
  • Aplicar restricciones de seguridad
  • Mantener la integridad
  • Instalación costosa
  • Baja estandarización
  • Personal especializado

Un sistema de bases de datos es la combinación de base de datos y software DBMS.

Motor de bases de datos

El motor de bases de datos es el corazón del sistema. Es el componente central del sistema de base de datos que realiza las operaciones fundamentales como la creación, lectura, actualización y eliminación de datos (CRUD).

Es responsable de:

  • La ejecución de consultas SQL.
  • La gestión del almacenamiento y recuperación de datos.
  • La aplicación de restricciones de integridad.
  • El control de transacciones y el manejo de la concurrencia.

Gestión de bases de datos

Los Sistemas de Gestión de Bases de Datos o DBMS permiten a los administradores o usuarios crear, editar y manipular datos dentro de una base de datos de manera segura y eficiente. Aseguran la integridad de los datos mediante el control del acceso, la gestión de las transacciones, el respaldo y la recuperación de la información, protegiendo los datos contra pérdidas o daños. Proporcionan seguridad, integridad y disponibilidad, que son características necesarias dentro de una base de datos.

El DBMS es un conjunto completo de software que incluye el motor de base de datos y otras herramientas y servicios necesarios para gestionar la base de datos. Un DBMS proporciona:

  • Herramientas de administración y configuración.
    • Definición de bases de datos usando DDL.
    • Manipulación de bases de datos usando DML.
  • Integridad de datos.
  • Permite transacciones: conjuntos de sentencias que se ejecutan como unidad.
  • Interfaces para usuarios y administradores.
  • Mecanismos de seguridad y control de acceso - DCL.
  • Funcionalidades de backup y recuperación.
  • Soporte para lenguajes de programación y APIs.
  • Funciones avanzadas de optimización y tuning.

El DBMS es un sistema de software de propósito general que facilita los procesos de definición, construcción, manipulación y compartición de bases de datos entre varios usuarios y aplicaciones.

Definir una base de datos implica especificar los tipos de datos, estructuras y restricciones de los datos que se almacenarán en la base de datos. La definición o información descriptiva de una base de datos se almacena en forma de catálogo o diccionario de la base de datos; es lo que se conoce como metadatos.

La construcción de la base de datos es el proceso consistente en almacenar los datos en algún medio de almacenamiento controlado por el DBMS. La manipulación de una base de datos incluye funciones como la consulta de la base de datos para recuperar datos específicos, actualizar la base de datos para reflejar los cambios introducidos y generar informes a partir de los datos. Compartir una base de datos permite que varios usuarios y programas accedan a la base de datos de forma simultánea.

Una aplicación accede a la base de datos enviando consultas o solicitudes de datos al DBMS. Una consulta normalmente provoca la recuperación de algunos datos; una transacción puede provocar la lectura o la escritura de algunos datos en la base de datos.

Otras funciones importantes ofrecidas por el DBMS son la protección de la base de datos y su mantenimiento durante un largo periodo de tiempo. La protección incluye la protección del sistema contra el funcionamiento defectuoso del hardware o el software (caídas) y la protección de la seguridad contra el acceso no autorizado o malintencionado. Una gran base de datos típica puede tener un ciclo de vida de muchos años, por lo que el DBMS debe ser capaz de mantener el sistema de bases de datos permitiendo que el sistema evolucione según cambian los requisitos con el tiempo.

No es necesario utilizar software DBMS de propósito general para implementar una base de datos. Podríamos escribir nuestro propio conjunto de programas para crear y mantener la base de datos; podríamos crear nuestro propio software DBMS de propósito especial. En cualquier caso (utilicemos o no un DBMS de propósito general), normalmente tenemos que implantar una cantidad considerable de software complejo. La mayoría de los DBMS son sistemas de software muy complejos.

Un motor de base de datos (database engine) y un sistema de gestión de bases de datos (DBMS, Database Management System) no son exactamente lo mismo, aunque están estrechamente relacionados.

El motor de base de datos es el núcleo que se encarga de las operaciones de manejo de datos. El DBMS incluye el motor de base de datos y otras herramientas y servicios para facilitar la administración y el uso de la base de datos. Ejemplos de DBMS: MySQL, PostgreSQL, Oracle, Microsoft SQL Server, todos los cuales contienen un motor de base de datos como componente central.

Tipos de bases de datos

Diferentes necesidades requieren diferentes tipos de bases de datos.

Las bases de datos relacionales organizan los datos en tablas interconectadas permitiendo relaciones complejas entre ellas. Dentro del mundo de las bases de datos relacionales existen varias opciones, como Oracle, SQL Server o MySQL, que es una de las bases de datos más populares a nivel mundial y también es sencilla en su uso.

Las bases de datos no relacionales o NoSQL, como MongoDB, son ideales para datos no estructurados, como imágenes, audios o archivos. Ofrecen flexibilidad en el almacenamiento y consulta de datos.

También existen las bases de datos cloud o en la nube, como la de Amazon o la de Google, que ofrecen escalabilidad, es decir que pueden aumentar fácilmente su tamaño, y también accesibilidad, permitiendo a los usuarios acceder a la información en cualquier lugar y en cualquier momento.

Base de datos relacional

Es un sistema sofisticado y organizado de almacenamiento de datos que utiliza tablas para organizar la información en filas y columnas permitiendo búsquedas rápidas y análisis complejos.

Componentes de una base de datos relacional

Una base de datos relacional está compuesta por un conjunto de tablas. Cada una de estas tablas almacena registros de datos en filas. Cada fila representa un único conjunto de información relacionada. Los campos o columnas categorizan esta información en tipos específicos, como textos, números o fechas. Los registros son las unidades individuales de información.

Por ejemplo: si tenemos una tabla de Alumnos, cada fila de la tabla se va a corresponder con los datos de un alumno determinado. Cada una de las columnas será un dato que nos interesa registrar de los alumnos, como fecha de nacimiento, dirección, etc.

Las tablas se relacionan entre sí. Para acceder a los datos, se usa el Lenguaje de Consultas Estructuradas o SQL(Structured Query Language), que permite obtener y alterar datos de una forma organizada siempre y cuando tengamos en cuenta cuál es la estructura de la base de datos con la que estamos trabajando. Para ello se utilizan los distintos comandos que SQL pone a disposición.

Las tablas de una bases de datos relacional se organizan a través de identificadores. Cada tabla tiene un identificador único que es el que va a establecer su relación con el resto de tablas. Estos identificadores hacen que sea más fácil organizar cada una de las tablas por separado.

Los DBMS relacionales se caracterizan por modelar la información en tablas que se relacionan entre sí. Ejemplos son MySQL, MariaDB, PostgreSQL, SQLServer, Oracle, entre otros.

Base de datos no relacional

Las bases de datos no relacionales están diseñadas para modelos de datos con estructuras más específicas y que no necesitan ser relacionados con otros. Cada entidad funciona de forma independiente y son mucho más sencillas que las relacionales.

La sencillez de acceso y ordenación las hace útiles para Big Data, ya que al no tener una estructura definida como las relacionales, se puede tener redundancia de datos, es decir, podemos tener datos repetidos.

Se permite la redundancia porque se prioriza el acceso rápido por sobre la normalización e integridad de los datos. En este tipo de modelos se requiere ahorrar poder computo para poder procesar la mayor cantidad de datos en el menor tiempo posible.

Las bases de datos no relacionales pueden tener identificador único, aunque no sea necesario para relacionar unos registros con otros. La información se organiza normalmente mediante documentos y es muy útil cuando no tenemos un esquema exacto de lo que se va a almacenar.

El formato más utilizado en las bases de datos no relacionales es el de un documento, que es un objeto con una clave y un valor para que el acceso a la información se pueda realizar de forma sencilla.

A todo este conjunto de bases de datos no relacionales, también se les dice NoSQL o Not Only SQL, para hacer referencia a que la información de un sistema no sólo puede almacenarse en modelos relacionales.

Los DBMS no relacionales modelan la información de diferentes formas: documentos, llave-valor, grafos, entre otras. Ejemplos de DBMS no relacionales son MongoDB, Redis, Apache Cassandra, Firebase entre otras.

¿Cuándo utilizar SQL o NoSQL?

Ambos tipos de bases de datos son útiles y dependerá del tipo de aplicación que queramos realizar la elección de una u otra base de datos.

Si queremos desarrollar una aplicación de tipo contable, de inventario o de información de clientes, es probable que el modelo relacional se adapte mejor a nuestras necesidades. En este tipo de aplicaciones, normalmente habrá más de una tabla que tenga relación con el resto, por lo que una base de datos relacional será más útil y podrá representar mejor nuestra aplicación.

Si nuestra aplicación necesita de un sistema en el que los datos que se vayan a almacenar no necesitan relacionarse unos con otros, y además no tenemos certeza de que todos los datos tengan la misma estructura, entonces usaremos una base de datos no relacional. Por ejemplo puede ser una base de datos en la que solo queramos almacenar las estadísticas de comportamiento de un usuario al visitar un sitio, o una base para recolectar sus datos biométricos, esta información puede cambiar de una persona a otra, dependiendo de sus dinámicas con el sitio o de su estado de salud y condición física para los biométricos. Otro ejemplo sería la creación de una galería de fotos en Facebook o Instagram, cada usuario puede subir la cantidad de fotos que guste sin que exista una estructura determinada. Las estadísticas de progreso de un videojuego también serían un buen ejemplo de modelo no relacional, ya que la información de un jugador a otro puede variar.

Todos estos ejemplos almacenan la información y no la relacionan entre sí.

La elección del tipo de base de datos es algo muy importante y que no hay que tomarlo a la ligera. La elección de un DBMS erróneo puede traer consecuencias fatales en el desarrollo de un proyecto, por lo que es muy importante dedicarle tiempo y ver qué tipo se adapta mejor con el modelo de nuestro proyecto.

La principal diferencia entre las bases de datos SQL y NoSQL es su estructura de almacenamiento de datos.

Las bases de datos SQL utilizan una estructura organizada y relacional, mientras que las bases de datos NoSQL utilizan una estructura más flexible y escalable. Ambas tienen sus propias fortalezas y debilidades, y la elección de una depende de las necesidades específicas de la aplicación y el tipo de datos que deben almacenarse.

Consideraciones a tener en cuenta al elegir entre un modelo de base de datos SQL y NoSQL:

    SQL
  • Relaciones estructuradas: Si la aplicación depende de relaciones estructuradas y complejas entre los datos, conviene considerar una base de datos SQL ya que permiten modelar relaciones complejas entre tablas y utilizar lenguajes de consulta estandarizados para realizar consultas complejas.
  • Datos estructurados: Si los datos tienen una estructura bien definida y es poco probable que cambie con frecuencia, una base de datos SQL es una buena opción, ya que permite definir esquemas rigurosos que garantizan la integridad de los datos y evita la introducción de datos erróneos o inconsistentes.
    NoSQL
  • Escalabilidad: Si la aplicación está destinada a crecer rápidamente y manejar una gran cantidad de datos, conviene considerar una base de datos NoSQL, que son más escalables que las bases de datos SQL y se adaptan mejor a grandes cantidades de datos no estructurados.
  • Datos no estructurados: Si los datos no tienen una estructura bien definida o cambian con frecuencia, conviene considerar una base de datos NoSQL, que son más flexibles que las bases de datos SQL y permiten modelar datos de manera más natural.
  • Aplicaciones en tiempo real: Si la aplicación requiere una respuesta rápida y en tiempo real, conviene considerar una base de datos NoSQL, que suelen ser más eficientes que las bases de datos SQL en términos de rendimiento y escalabilidad en aplicaciones en tiempo real.

La elección entre un modelo de base de datos SQL o NoSQL depende de las necesidades específicas de la aplicación y de la naturaleza de los datos. Es importante evaluar cuidadosamente los requisitos y considerar las fortalezas y debilidades de cada opción antes de tomar una decisión.
