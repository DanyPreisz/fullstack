LeetCode: 1. Two Sum

Comprobar si existe un par de números con una suma dada en un array

Declaración del problema: Dado un array de enteros nums[] y un entero objetivo o target, retornar los índices de dos números tales que su suma es igual al target.

Nota: Asumir que existe una sola solución, y que no se puede utilizar el mismo elemento dos veces. Ejemplo: Si target es igual a 6 y nums[1] = 3, entonces nums[1] + nums[1] = target no es una solución.


Brute Force Solution

Código



La función twoSum() se utiliza para encontrar dos números en un arreglo que sumen un valor específico (llamado "target"). La función toma dos parámetros: un arreglo de números llamado "nums" y un entero llamado "target". La función devuelve un arreglo con los índices de los dos números que suman "target", o "undefined" si no se encuentra ningún par que cumpla esta condición.

La función comienza con un ciclo que itera sobre los elementos del arreglo "nums" (línea 2). Dentro de este ciclo, se utiliza otro ciclo para iterar sobre todos los elementos del arreglo "nums" que se encuentran a la derecha del elemento actual (línea 3). Si se encuentra un par de elementos que sumen "target", se devuelve un arreglo con sus índices y se sale de la función (línea 4). Si se terminan los ciclos sin encontrar ningún par que cumpla esta condición, se devuelve "undefined".

Este código tiene una complejidad temporal de O(N^2), ya que utiliza dos ciclos anidados para iterar sobre todos los pares posibles de números en el arreglo. Esto significa que el tiempo de ejecución del código crece de manera cuadrática con la cantidad de números en el arreglo. Por lo tanto, este código no es muy eficiente para arreglos grandes.


Pseudocódigo


El siguiente algoritmo implementa la solución del problema "Two Sum" en un lenguaje de programación orientado a procedimientos. Este algoritmo busca parejas de números en un arreglo de enteros que, al sumarse, den como resultado un valor determinado (target). El algoritmo comienza pidiendo al usuario que ingrese el valor de target y la cantidad de números que se van a evaluar. Luego, se piden al usuario que ingrese los números uno por uno y se almacenan en el arreglo "nums". Luego, el algoritmo itera sobre cada elemento del arreglo "nums" y muestra en pantalla su índice y su valor. Después, se itera nuevamente sobre cada elemento del arreglo "nums", comparando cada elemento con todos los demás elementos que vienen después de él en el arreglo. Si se encuentra una pareja de elementos que, al sumarse, den como resultado el valor de target, se muestra en pantalla el valor de la pareja de elementos y sus índices en el arreglo. Este algoritmo utiliza dos ciclos anidados (un "Para" dentro de otro "Para") para recorrer todos los elementos del arreglo y compararlos entre sí. Esta es una técnica conocida como búsqueda de fuerza bruta, ya que compara todas las posibles parejas de elementos para encontrar la solución. En general, este tipo de algoritmos son eficientes para problemas pequeños, pero pueden tardar mucho tiempo en resolver problemas más grandes.
