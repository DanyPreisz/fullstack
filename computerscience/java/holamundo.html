Primer programa: "Hola Mundo"

Como iniciación, vamos a escribir un programa "Hola mundo" muy sencillo. Con esto conseguiremos familiarizarnos con algunas de las características del lenguaje Java y vamos a lograr nuestro primer programa funcional. Vamos a escribir nuestro código en un editor de textos, como el Bloc de notas. Es muy importante que no usemos un procesador de textos porque este añade imformación que el compilador de Java no será capaz de interpretar.

Todo el código de Java debe de estar encapsulado dentro de una clase. En nuestro caso, nuestra clase se llama HolaMundo. Por convención, el nombre del archivo que aloja el código fuente debe de ser el mismo que el nombre de la clase principal contenida en ese archivo. La extensión de ese archivo debe de ser .java, por lo que vamos a guardar como HolaMundo.java.

Los programas Java necesitan un método main para que el intérprete de Java sepa por dónde tiene que comenzar a ejecutar el código. El método main es la puerta de entrada a nuestro código.


Compilación

Ahora que ya tenemos el archivo con nuestra clase HolaMundo, que contiene un método main, ya podemos compilarlo. Vamos a compilarlo por línea de comandos usando las herramientas del kit de desarrollo de Java, el JDK, que debemos tener instalado. El compilador de Java se llama javac. Para compilar nuestro programa, vamos a un terminal a la ruta donde tenemos guardado nuestro archivo. Para ello, lo más fácil es, en el gestor de archivos, pulsar en el path y escribir cmd, pulsar Enter, y se abre el terminal. Para compilar por línea de comandos, escribimos javac y el nombre de nuestro archivo, HolaMundo.java. Pulsamos Enter y, ahora, lo que ha ocurrido es que se nos ha creado el archivo HolaMundo.class. Ese archivo contiene el bytecode que ejecutará la máquina virtual de Java.

Ejecución

Para ejecutar el programa, debemos usar el intérprete de Java con el comando java. Vamos de nuevo a la consola y escribimos java HolaMundo, que es el nombre de nuestra clase con un método main. Vemos que lo que nos ha hecho este método es escribir por pantalla Hola Mundo. En cada archivo .java puede haber más de una clase. Si en el archivo HolaMundo.java hubieran habido más clases, al compilar se habría generado un archivo .class para cada una de ellas. Al intérprete de Java debemos pasarle el nombre de la clase que queremos que ejecute, en este caso, HolaMundo. Y esta clase debe tener un método main para que el intérprete sepa por dónde comenzar a ejecutar.

Comentarios

Son líneas que el compilador va a ignorar, y son informativos para el programador.

Java soporta tres tipos de comentarios:

  • Comentarios de una línea, como el que dice: // Un programa Java necesita un método main por el que empezar. Comienzan con dos barras.
  • Comentarios de varias líneas, como el que podemos ver al inicio del programa. Comienzan por barra/asterisco, y terminan por asterisco/barra.
  • Comentarios de documentación, que son como los comentarios multilínea, pero tienen dos asteriscos al inicio. Sirven para la herramienta Javadoc para que pueda generar documentación automáticamente.

Análisis del código de nuestro primer programa:

  • Declaración de la clase. Todo el código de Java debe de estar encapsulado dentro de clases. Usamos la palabra clave class para declarar la clase, y tras ella, ponemos el nombre de la clase, en este caso, HolaMundo. Por convención, los nombres de clases en Java comienzan con letra mayúscula.
  • Definición de la clase. Todos los miembros de la clase irán dentro de corchetes.
  • La palabra clave public es un modificador de acceso, y esto quiere decir que esta clase es accesible desde cualquier otra clase.
  • El método main. Todos los programas Java necesitan uno y será el lugar donde comience a ejecutarse el programa.
  • La palabra clave static permite que el método main se llame sin necesidad de que un objeto de esta clase haya sido creado.
  • La palabra clave void quiere decir que este método no devuelve nada.
  • Toda la información que queremos pasarle a un método tiene que ir en parámetros dentro de los paréntesis que van tras su nombre. En nuestro programa se le pasa al método main una serie de secuencias de caracteres que pueden ser introducidas en el momento de ejecución por la línea de comandos. En este caso, no estamos haciendo uso de ellos.
  • El método println del flujo de salida out de la clase System. System es una clase que ya viene predefinida en Java. Lo que hace println es sacar por out, que es la salida conectada a la consola, el parámetro que le indiquemos entre paréntesis, en este caso, el string Hola Mundo. Esta línea es la única sentencia que tenemos en el programa. Es la única orden que estamos dando, la única acción que se está produciendo. Las sentencias terminan con un punto y coma.

Generalmente, ningún desarrollador escribe, compila y ejecuta el código de este modo, por consola, sino que se ayudan de un IDE, que son las siglas en inglés de entorno integral de desarrollo, y son programas que tienen muchas herramientas para facilitarnos el desarrollo, pero está muy bien saber el modo básico de hacerlo.
