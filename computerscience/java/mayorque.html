Segundo programa: "Mayor que"


Análisis del código


Java es un lenguaje bastante sencillo y resulta bastante natural.



  • Creamos la clase MayorQue y le añadimos el método main().
  • Creamos dos variables de tipo entero y les asignamos un valor.
  • Si variable1 es mayor que variable2 se ejecutan las instrucciones dentro de las primeras llaves.
  • Si no es mayor se ejecuta el código que está dentro de las segundas llaves.
  • Al ejecutar el programa aparece una ventana que nos muestra el mensaje "variable2 es mayor que variable1". Como variable1, que contiene el valor 5, no es mayor que variable2, que contiene el valor 10, no se ha ejecutado el código que viene tras el If, sino el que viene tras el Else.

Normas de estilo

En la estructura hay diferentes líneas con diferentes indentaciones o sangrías. Java no necesita estos espacios para entender nuestro programa, sino que se ponen para que luego sea más fácil de leer o interpretar por otra persona o por ti mismo. Es una convención a la que se ha llegado a lo largo de muchos años y que siguen también otros lenguajes de programación. Como norma general, añadiremos un sangrado extra en la siguiente línea a la apertura de una llave. Las llaves de apertura se escriben acompañando a la primera línea, mientras que las llaves de cierre se escriben solas en una línea aparte. Conviene conocer las normas de estilo para mantener un lenguaje universal.


Identificadores y palabras claves

Un identificador es una secuencia de caracteres que el desarrollador elige para identificar diferentes elementos creados por él. Un identificador es el nombre que le damos a las variables, a los métodos, a las clases, a los paquetes y a los interfaces.

Para que sea válido para Java, un identificador debe comenzar con una letra, un guion bajo o un carácter de moneda. Si esto no se cumple, nuestro programa no compilará. Si no estamos seguros de si un carácter en particular es válido para usarse para comenzar un identificador podemos usar el método isJavaIdentifierStart, de la clase Character, que nos devolverá verdadero o falso.

Un identificador puede contener letras, números, caracteres de moneda o caracteres de conexión. No es obligatorio que contenga todos estos tipos, pero todos los caracteres deben ser de estos tipos. Si no estamos seguros de si un carácter en particular es válido para usarse para formar un identificador, para comprobarlo tenemos el método isJavaIdentifierPart, de la clase Character, que nos devolverá también true o false.

La longitud mínima de un identificador es de un carácter y no tiene longitud máxima.

No puede utilizar ninguna palabra clave, pero sí puede contenerla.

Los identificadores son case-sensitive; se diferencia entre mayúsculas y minúsculas.

Las convenciones son una serie de reglas universales no obligatorias que ayudan a la legibilidad de nuestro programa.

Para las clases interfaces, se usa el formato UpperCamelCase. Esto quiere decir que se comienza con mayúscula y también son mayúsculas la primera letra de cada palabra interna. Se recomienda usar palabras completas, no abreviaciones ni acrónimos, a no ser que los acrónimos sean ampliamente usados. Si vamos a usar acrónimos, solo la primera letra estará en mayúsculas.

Para los métodos, usamos el formato lowerCamelCase. Se comienza en minúscula, pero se escribe en mayúscula la primera letra de cada palabra interna. Los identificadores deberán representar acciones mediante verbos en infinitivo o imperativo. En inglés no existe el problema porque el imperativo y el infinitivo son iguales.

Para las variables, igual que para los métodos, usamos lowerCamelCase, por lo tanto, comienzan con minúscula, pero será mayúscula la primera letra de cada palabra interna. Debemos elegir nombres descriptivos. No importa que sean largos, porque los IDE ofrecen ayuda de autocompletado. No conviene usar nombres de un solo carácter, excepto para variables temporales. Ejemplos de nombres de variables: clienteActual, velocidad o numeroDePedido. Podemos usar acentos para nuestros identificadores pero, por hacerlos más universales, en ocasiones se decide no usarlos.

Para las constantes, usamos todo mayúsculas, con las palabras separadas por guion bajo. Por ejemplo, PI o NUMERO_DE_SEMANAS_ANUALES.

Las palabras clave están reservadas por Java y no pueden ser usadas para nuestros identificadores. Las palabras true, false y null, que no son palabras clave, sino que son literales, tampoco pueden ser usadas como identificadores.
